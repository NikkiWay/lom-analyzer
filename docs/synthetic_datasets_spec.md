# Спецификация синтетических датасетов для LOM Analyzer

## 1. Темы

### Тема A: ИИ и нейросети в России
Технологическая повестка. Конкуренция нарративов: корпорации (Сбер, Яндекс) продвигают свои продукты, эксперты предупреждают о рисках, пользователи делятся опытом, СМИ усиливают сигнал. Разные позиции: от восторга до страха замены профессий.

### Тема B: Продвижение мессенджера Max (VK)
Корпоративная информационная кампания внутри платформы. VK продвигает Max как замену Telegram/WhatsApp. Акторы: официальные аккаунты VK, блогеры-промоутеры (возможно оплаченные), скептики, обычные пользователи. Позволяет выявить разницу между органическими и управляемыми лидерами мнений.

### Тема C: Отказ от Болонской системы в вузах
Государственная реформа: возврат к специалитету. Минобрнауки продвигает, ректоры комментируют, преподаватели-блогеры спорят, студенты обсуждают судьбу своих дипломов, работодатели высказывают позицию. Конфликт нарративов: государство vs академическое сообщество vs студенты.

## 2. Технические требования к данным

### 2.1. Формат файла

JSON в формате `ImportDataset`:

```json
{
  "communities": [...],
  "authors": [...],
  "posts": [...],
  "comments": [...]
}
```

### 2.2. Минимальные объёмы (на один датасет)

| Сущность | Минимум | Рекомендовано | Обоснование |
|---|---|---|---|
| Авторы | 30 | 50-80 | Bootstrap B=1000 требует >= 20 наблюдений для стабильных CI |
| Посты CURRENT (тематические) | 100 | 200-400 | TopVol, TopFocus, Reach, Pos, ER_top |
| Посты BASELINE (фоновые) | 150 | 300-500 | ER_bg, Age, TopFocus (знаменатель) |
| Комментарии | 50 | 100-300 | Resp_a (двухуровневый бутстрап 300x100) |
| Сообщества | 2 | 3-5 | Контекст, источник постов |

### 2.3. Обязательные поля и питаемые метрики

**Авторы (`ImportAuthor`):**

| Поле | Тип | Метрика |
|---|---|---|
| `vkId` | Int | Связка с постами через fromId |
| `firstName`, `lastName` | String | UI, экспорт |
| `screenName` | String | UI |
| `followersCount` | Int | Aud = ln(1+F), ER_bg, ER_top |
| `isClosed` | Boolean | Фильтрация закрытых аккаунтов |

**Посты (`ImportPost`):**

| Поле | Тип | Метрика |
|---|---|---|
| `vkId` | Int | Уникальность |
| `ownerId` | Int | Связка с сообществом (отрицательный) или автором (положительный) |
| `fromId` | Int | Связка с автором |
| `date` | Long (unix) | Попадание в BASELINE/CURRENT окно |
| `text` | String | Тематическая фильтрация, NLP-тональность, дедупликация |
| `likes` | Int | ER_bg, ER_top (L+C+R) |
| `reposts` | Int | ER_bg, ER_top (L+C+R) |
| `comments` | Int | ER_bg, ER_top (L+C+R); фильтр для сбора комментариев |
| `views` | Int | Reach = sum(V_i) |
| `window` | String | "CURRENT" или "BASELINE" |
| `containsMedia` | Boolean | Доп. аналитика |
| `hasCopyHistory` | Boolean | Дедупликация (репосты) |

**Комментарии (`ImportComment`):**

| Поле | Тип | Метрика |
|---|---|---|
| `vkId` | Int | Уникальность |
| `postVkId` | Int | Связь с постом |
| `postOwnerId` | Int | Связь с постом |
| `fromId` | Int | Автор комментария |
| `date` | Long (unix) | Временная привязка |
| `text` | String | Resp_a (тональность комментариев) |
| `likes` | Int | Вес комментария |

### 2.4. Требования к распределению данных

**Авторы — разнообразие followersCount:**
- 5-10 авторов с высокой аудиторией (10 000 - 500 000+) -> потенциальные Гиганты/Лидеры
- 15-25 авторов со средней аудиторией (500 - 10 000) -> Активисты
- 15-30 авторов с малой аудиторией (10 - 500) -> Фон
- Несколько закрытых аккаунтов (isClosed = true) -> проверка фильтрации

**Посты — разнообразие активности:**
- У топ-авторов: 5-15 тематических постов (CURRENT) + 10-30 фоновых (BASELINE)
- У средних: 2-5 тематических + 5-15 фоновых
- У фоновых: 1-2 тематических + 3-5 фоновых
- Реакции пропорциональны аудитории автора
- views > likes (views = likes * 15-50)

**Комментарии — тональность:**
- Под постами топ-авторов: 5-20 комментариев
- Под постами средних: 1-5 комментариев
- Разнообразие: позитивные, негативные, нейтральные
- Комментарии от разных fromId (не только от авторов из реестра)

**Временные окна (поле `date`, unix timestamp):**
- CURRENT: последние 30 дней от момента генерации
- BASELINE: от 90 до 30 дней назад
- Посты распределены по разным дням внутри окна

### 2.5. Ожидаемые роли (квадранты)

Каждый датасет должен порождать все 4 роли:

| Роль | Условие | Типичный автор |
|---|---|---|
| Лидер | Struct > median AND Topic > median | Большая аудитория + много тематических постов |
| Активист | Struct < median AND Topic > median | Малая аудитория, но активен по теме |
| Гигант | Struct > median AND Topic < median | Большая аудитория, редко пишет по теме |
| Фон | Struct < median AND Topic < median | Малая аудитория + мало постов |

Плюс атрибуты:
- Позиция автора (Pos_a): POSITIVE / NEUTRAL / NEGATIVE
- Отклик аудитории (Resp_a): распределение тональности комментариев

### 2.6. Правдоподобные соотношения метрик

| Соотношение | Диапазон | Пример |
|---|---|---|
| views / likes | 15-50 | 10 000 просмотров -> 200-700 лайков |
| likes / comments | 3-20 | 200 лайков -> 10-70 комментариев |
| likes / reposts | 3-10 | 200 лайков -> 20-70 репостов |
| views у сообществ vs личных страниц | 5-20x | Пост в группе 50 000 просмотров vs личный 3 000 |

## 3. Требования к реалистичности (КРИТИЧЕСКИ ВАЖНО)

### 3.1. Авторы

**Обязательный процесс при генерации:**
1. Провести поиск в интернете и ВКонтакте по теме датасета
2. Найти реальных публичных авторов, которые высказываются по данной теме
3. Использовать их реальные имена, screenName, приблизительные followersCount
4. Для публичных персон (блогеры, эксперты, журналисты, чиновники) -- брать реальные данные
5. Для обычных пользователей -- создавать правдоподобные русские ФИО с реальными фамилиями

**Запрещено:**
- Говорящие/шаблонные фамилии: "Иванов", "Провокаторов", "Датасетов", "Репостова", "Нейронова"
- Очевидно сгенерированные screenName: "ai_lover_42", "test_user_7", "neural_expert"
- Одинаковые followersCount у разных авторов
- Авторы без связи с темой
- Нереальные VK ID (использовать диапазон реальных ID: 1 000 000 - 800 000 000)

**Допустимо:**
- Использовать реальные имена публичных лиц, если они действительно публично высказывались по теме
- Создавать вымышленных авторов с правдоподобными ФИО (обычные русские фамилии: Кузнецов, Морозова, Лебедев и т.д.)
- Слегка округлять followersCount реальных авторов (+-10%)

### 3.2. Тексты постов

- Стиль соответствует типу автора: эксперт пишет развёрнуто и со ссылками, обычный пользователь -- коротко и разговорно, СМИ -- формально
- Тематические посты содержат ключевые слова для прохождения фильтра
- Фоновые посты (BASELINE) -- о другом: быт, работа, хобби, нерелевантные темы
- Несколько намеренно off-topic постов для тестирования фильтрации
- Тексты не должны быть шаблонными или выглядеть как сгенерированные ИИ: без "в мире, где...", "важно отметить", "в заключение хочется сказать"

### 3.3. Комментарии

- Короткие, живые, с разговорной лексикой
- Включают сленг, сокращения, опечатки (как в реальных VK-комментариях)
- Разная тональность: согласие, несогласие, вопросы, благодарность, критика, троллинг
- Не все комментарии по теме -- часть может быть бытовыми репликами

## 6. Процесс генерации каждого датасета

1. **Исследование темы** -- поиск в интернете и VK: кто реально обсуждает, какие сообщества, какие позиции
2. **Составление реестра авторов** -- реальные публичные персоны + правдоподобные вымышленные
3. **Генерация сообществ** -- на основе реальных VK-сообществ по теме
4. **Генерация постов** -- тексты, стилистически соответствующие авторам, с реальными ключевыми словами
5. **Генерация комментариев** -- живые реплики под тематическими постами
6. **Валидация** -- проверка соотношений метрик, полноты данных для всех 11 формул
