# Архитектура прототипа LOM Analyzer

Связь модулей с подразделом 2.2.3 диплома.

## Четыре модуля

| Модуль | Пакет | Назначение | Подраздел |
|---|---|---|---|
| UI | `ui/` | Compose Desktop, 7+ экранов | 2.2.6 |
| Сбор данных | `vk/`, `import/` | VK API, OAuth, комментарии, импорт | 2.2.4, 2.2.5 |
| NLP | `nlp/`, `preprocessing/` | Двухрежимный: Python sidecar / Kotlin fallback | 2.2.7 |
| Аналитическое ядро | `analysis/` | 11 оценок, бутстрап, композиты, роли, качество | 2.1.3–2.1.6, 2.2.8 |

## Изоляция модулей

Модули обмениваются данными **только через SQLite** (режим WAL).
Никаких прямых вызовов между модулями — DI через Koin (`di/AppModule.kt`).

## Подпакеты analysis/

| Подпакет | Назначение | Подраздел |
|---|---|---|
| `scoring/` | 11 количественных оценок | 2.1.3, Е.4 |
| `inference/` | Робастная статистика, бутстрап | 2.1.4, 2.1.5, Е.1–Е.3 |
| `composite/` | z-нормализация, композиты, пороги | 2.1.6, Е.4.6 |
| `roles/` | Квадрантная классификация | 2.1.6 |
| `sufficiency/` | Индикатор достаточности данных | 2.1.5 |
| `quality/` | Индикаторы качества сессии | 2.2.8, Приложение Г |
| `topic/` | Тематическая фильтрация | 2.1.1 (этапы 5-6) |
| `dedup/` | Дедупликация | 2.1.1 (этап 5) |
| `content/` | Словарный сентимент (fallback) | 2.2.7 |

## База данных

SQLite WAL, Flyway миграции V1–V10.
Ключевые таблицы: `analysis_session`, `post`, `author`, `comment`, `lom_score`,
`bootstrap_interval`, `composite_score`, `session_threshold`, `author_role`,
`session_quality_indicator`, `session_event`, `pipeline_checkpoint`, `nlp_result`.
